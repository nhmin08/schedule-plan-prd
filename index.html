KONDA
일정 계획 기능

사용자가 여행을 계획하는 과정을 KONDA 안으로 가져옴으로써, 맥락 없이 노출되던 상품 추천을 '지금 내 여행에 필요한 것'으로 전환하고 일일 평균 결제 횟수를 높이는 기능입니다.

01 제품 목표

사용자 & 비즈니스 목표

사용자 목표: 한국 여행 일정을 한 곳에서 빠르게 계획하고, 내 일정에 맞는 숙소·체험·교통을 번거롭게 찾지 않아도 추천받는다.

비즈니스 목표: 여행 계획이라는 고관여 맥락 안에서 상품을 노출함으로써 일일 평균 결제 횟수(북극성 지표)를 높이고, 신규 카드 발급 전환을 증가시킨다.
North Star Metric
사용자가 KONDA로 발생시키는
일일 평균 결제 횟수
Daily Transactions per Active User · 세 변수의 곱으로 분해됨
일일 결제 횟수 = 결제 활성 사용자 수 × 사용자당 결제 빈도 × 세션당 결제 전환율 일정 기획 기능은 이 세 변수 중 결제 빈도와 전환율에 직접 영향을 주고, 온보딩 단순화·일정 공유를 통해 활성 사용자 수에도 간접 기여합니다.
02 문제 정의

전환 이탈 원인 분석(가설)

홈·리뷰 페이지 체류 시간(평균 6.8분, 업계 평균 3.2분 대비 +113%)은 충분히 높습니다. 그러나 카드 발급 전환율은 1.2%에 그칩니다. 원인은 단일하지 않습니다. 이탈 패턴을 분해하면 세 가지 구조적 시나리오가 동시에 작동하고 있습니다.

A
즉시 이탈형 — 앱 설치 직후 이탈
기능을 둘러보기 전에 나간다. 추정 비중 62%. 온보딩 완료율 12%.
퍼널 1→2 구간
근본 원인: 진입 직후 화면이 카드 상품 나열 중심이라 "이 앱이 나에게 왜 필요한가"를 8초 안에 납득시키지 못합니다. 사용자의 현재 문제(여행 계획)와 연결고리가 없습니다.
측정 근거: 온보딩 완료율 12%(업계 평균 34%), 첫 화면 평균 체류 8초.
온보딩 완료율 현재 12% → 목표 40% 일정 기능 진입점 단순화로 해결
B
탐색 후 이탈형 — 상품 조회 후 구매 미전환
설명을 읽었지만 결제하지 않는다. 상품 조회 후 이탈율 78%.
퍼널 3→5 구간
근본 원인: 상품은 이해했지만 "지금 내가 왜 이걸 사야 하는가"의 맥락이 없습니다. 여행 날짜 없이 노출된 숙소 목록은 막연하고, 카드 혜택도 당장 쓸 여행이 없으면 발급 동기가 생기지 않습니다.
측정 근거: 상품 상세 평균 체류 2.3분(충분히 읽었으나 행동 없음), 이탈 직전 검색어 34%가 "일정·코스·추천" 키워드.
세션 전환율 현재 1.2% → 목표 3.5% 일정 맥락 위 상품 추천으로 해결
C
저체류 전환형 — 결제 후 앱 즉시 이탈
구매는 하지만 체류 시간 1.9분, 7일 재방문율 22%에 그친다.
결제 빈도 문제
근본 원인: 예약 완료 후 KONDA에 계속 있을 이유가 없습니다. 사용자는 네이버 지도·구글맵으로 이동해 나머지 여행을 계획합니다. 앱이 "예약 채널"로만 인식됩니다.
측정 근거: 결제 완료 사용자 평균 세션 1.9분, 7일 재방문율 22%(목표 40% 대비 절반 수준). 월 평균 결제 횟수 1.3회.
7일 재방문율 현재 22% → 목표 40% 예약→일정 자동 연동, 리마인더로 해결

05 리스크 및 대응 계획

주요 리스크 & 대응 방안

H
추천 알고리즘 정확도 부족
관련성 낮은 추천이 반복될 경우 사용자 신뢰 하락, 이탈률 증가로 전환율이 오히려 하락할 수 있습니다.
감지 기준: 추천 CTR < 15% 또는 NPS 주간 점수 < 30 시 알람.
완화 플랜: 출시 초기 룰 기반(Rule-based) 추천으로 시작. CTR 35% 달성 후 ML 모델 전환. A/B 테스트로 정확도 주 단위 측정.
플랜 B: ML 전환 지연 시 추천 결과 다양성 확대로 대체.
H
복잡한 UX로 인한 이탈 증가
기능 과부하가 오히려 시나리오 A 이탈을 심화시킬 수 있습니다. 일정 기능은 현재 최단 진입 경로가 5단계 이상입니다.
감지 기준: 온보딩 완료율 < 25% 또는 일정 생성 이탈율 > 60%.
완화 플랜: MVP는 목적지 + 날짜 2단계 입력만으로 즉시 추천 결과 노출. P2 이후 기능 단계적 추가. 각 기능 추가 전 A/B 테스트 필수.
플랜 B: 이탈 구간 특정 후 해당 단계 제거 또는 스킵 허용.
M
경쟁 서비스 기능 우위
트리플·야놀자·네이버 지도는 이미 일정 기능이 성숙합니다. 단순 일정 기능으로는 차별화가 어렵습니다.
차별점: KONDA 카드 혜택과 여행 일정의 연동은 타 서비스가 복제할 수 없는 독점 포지션. "이 여행에서 ₩XX 절약" 시뮬레이션은 금융 데이터 없이 불가.
완화 플랜: 일정 기능 자체보다 카드-일정 연동 UX에 개발 리소스 집중.
M
개인정보 및 금융 데이터 처리
여행 패턴 + 소비 패턴 + 카드 혜택 분석은 민감 데이터 결합입니다. 처리 방식에 따라 규제 리스크 및 사용자 불신이 발생할 수 있습니다.
감지 기준: 개인정보 동의 거부율 > 30%.
완화 플랜: Privacy by Design 원칙 적용. 법무팀 검토 출시 전 완료. 데이터 수집 목적 UI 내 명확 고지. 필요 최소한 수집 원칙.
플랜 B: 금융 데이터 비연동 시 일반 여행자 혜택 카드 추천으로 대체.
